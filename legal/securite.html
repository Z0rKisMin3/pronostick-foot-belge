POLITIQUE DE SECURITE

Version 1.0 - 27 juin 2026

1. Objet

La présente politique décrit les mesures de sécurité recommandées pour l'application PronoStick Foot Belge.

2. Protection des comptes

L'application doit utiliser :

  • authentification sécurisée
  • mots de passe chiffrés si les mots de passe sont gérés directement
  • limitation des tentatives de connexion
  • protection contre les accès automatisés
  • mécanisme de réinitialisation sécurisé

3. Accès administrateur

Les accès administrateur doivent être limités aux personnes autorisées.

Chaque administrateur doit utiliser un compte personnel.

Les actions sensibles doivent être journalisées lorsque cela est techniquement possible.

4. Données

Les données doivent être protégées par :

  • connexion HTTPS
  • base de données sécurisée
  • sauvegardes régulières
  • droits d'accès limités
  • suppression des données inutiles
  • contrôle des dépendances techniques

5. Signalement des failles

Toute faille ou suspicion de faille peut être signalée à : etienne.depryck@gmail.com

Le signalement doit contenir :

  • description du problème
  • étapes de reproduction
  • impact constaté
  • capture d'écran si utile
  • coordonnées du déclarant si une réponse est souhaitée

6. Comportement interdit

Il est interdit de :

  • tester la sécurité sans autorisation
  • accéder aux comptes d'autres utilisateurs
  • extraire des données
  • contourner les protections
  • perturber le service
  • exploiter une faille

7. Gestion des incidents

En cas d'incident, le gestionnaire peut :

  • suspendre temporairement le service
  • réinitialiser des accès
  • corriger les données
  • informer les utilisateurs concernés
  • notifier l'Autorité de protection des données si le RGPD l'impose
  • déposer plainte si nécessaire

8. Développement

L'application est développée avec Claude Code (Opus 4.8).

Hermes intervient uniquement comme consultant externe et ne développe pas l'application.
